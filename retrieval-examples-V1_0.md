## 检索效果示例与分析 Version 1.0

本页记录基于真实终端日志的两个典型检索案例，帮助快速理解：

- **检索 / 向量化是正常工作的**
- 为什么有时会命中「看起来不相关」但在语义空间更接近的问题
- 如何通过改写问题或调参改善效果

---

### 示例一：问题偏“心理学”，命中的是社科书单

- **问题**：  
  > 肠道菌群的代谢物如何影响人体心理健康？

- **检索参数**：
  - `k = 5`（默认）
  - `--debug-retrieval`
  - `--with-sources`
  - `--preview-chars 200`

- **检索中间状态（摘要）**：
  - Top‑5 命中文档全部来自公众号 `缪斯的书屋_缪斯书单`，主题集中在：历史书单、社科经典、人类学、集体暴力政治等。
  - 这些文档片段中高频出现的语义包括：「情绪、抑郁、新常态、社会结构、人性、政治暴力」等。
  - 目标 PDF《肠道菌群代谢物大全（价值千金）》**没有出现在前 5 条命中里**。

- **模型最终回答**：
  - 根据当前引用的参考内容（社科类书籍介绍），模型无法从中提取出「肠道代谢物 → 心理健康」的具体机制，因此按照 Prompt 约束，输出：  
    > 根据当前资料无法回答。

- **分析**：
  - 问题中「心理健康」这一语义信号非常强，而向量检索底层使用的是通用 Embedding 模型（当前示例为 GLM Embedding）。
  - 在向量空间里，「心理健康 / 抑郁 / 情绪」这些概念与**社科/人类学/政治心理学**类书单文本更接近，而与一篇偏「代谢物列表 +通路」的科普文距离稍远。
  - 由于 `k=5` 较小，这次检索只返回了与「心理/情绪/社会结构」高度相关的 5 条社科文档，目标 PDF 掉在 Top‑5 之后，因此既没有进入大模型上下文，也自然无法被引用。

- **改进建议**：
  1. **调大检索的 k 值**：例如 `--k 20`，观察更大范围的命中文档里是否出现目标 PDF。
  2. **改写问题，使之更靠近目标文档语义**：  
     - 例如：  
       > 肠道菌群产生的短链脂肪酸、胆汁酸、多糖降解产物等代谢物，各自有哪些类型和功能？  
       这会更容易命中《肠道菌群代谢物大全（价值千金）》。
  3. 后续可考虑引入 **混合检索（BM25 + 向量）或 rerank**，对关键词高度匹配（如“肠道菌群代谢物”）的医学 PDF 给予额外加权。

### 示例二：问题更贴近目标 PDF，命中效果显著提升

- **问题**：  
  > 肠道菌群代谢物有哪些？

- **检索参数**：
  - `k = 5`
  - `--debug-retrieval`
  - `--with-sources`
  - `--preview-chars 200`

- **检索中间状态（摘要）**：
  - Top‑1 和 Top‑4 均为：  
    `肠道菌群代谢物大全（价值千金）`（公众号：肠道知识科普），不同页码的片段。
  - 预览片段中出现了非常明确的结构化信息：
    - 「短链脂肪酸（SCFAs）、乙酸、丙酸、丁酸、异丁酸、异戊酸、戊酸」  
    - 「胆汁酸代谢物：胆酸、鹅脱氧胆酸、脱氧胆酸、石胆酸」  
    - 「多糖降解产物：葡萄糖、果糖、半乳糖、琥珀酸」  
    - 「三甲胺 / TMAO、叶酸、维生素 K 等」

- **模型最终回答（摘要）**：
  - 系统性列出了：
    - **短链脂肪酸（SCFAs）**：乙酸、丙酸、丁酸等，来源菌群与核心生理功能；
    - **胆汁酸代谢物**：初级/次级胆汁酸及其来源与代谢作用；
    - **多糖降解产物**：葡萄糖、果糖、半乳糖、琥珀酸等；
    - **其他重要代谢物**：TMA/TMAO、叶酸、维生素 K 等；
  - 并在回答中用 `[1]`、`[4]` 等引用号标注，指向对应的 PDF 片段来源。

- **命中文档列表（摘要）**：
  - `[1]` / `[4]` 均为：  
    `F:\AIhub\WXdown-RAG\data\WXhub\肠道知识科普\pdf\2026-01-30-肠道菌群代谢物大全（价值千金）.pdf`
  - 其它命中多为与问题关系较弱的社科书单，但不会影响主答案的核心内容。

- **分析**：
  - 当问题直接聚焦于「肠道菌群代谢物有哪些？」时，语义空间更靠近目标 PDF 中的大量术语（SCFAs、胆汁酸、多糖降解产物、TMA/TMAO 等）。
  - 向量检索因此把这篇 PDF 提升到 Top‑1/Top‑4，成为主要的上下文来源。
  - 最终回答内容基本可以视为对该 PDF 表格内容的结构化复述，体现了 RAG 框架在「清单/分类」类问题上的优势。

### 总结：如何在实际使用中获得更稳定的检索效果？

- **1. 合理设置 k（检索条数）**
  - 对于简单问答，`k=5` 一般足够；
  - 对跨学科、多概念的问题（如「肠道代谢物 → 心理健康」），建议尝试 `k=10~20`，并搭配 `--debug-retrieval` 查看实际命中。

- **2. 尽量让问题语言贴近你真正关心的文档内容**
  - 如果你心里已有「目标文档」，可以在问题中包含这篇文档中的**关键术语 / 核心概念**，帮助 Embedding 更准确定位。

- **3. 充分利用调试输出**
  - 使用：

    ```bash
    python -m wxchatrag.cli query --question "你的问题" ^
      --with-sources ^
      --debug-retrieval ^
      --preview-chars 200
    ```

  - 先看「=== 检索中间状态 ===」中前几条命中是否合理，再看最终回答是否符合预期。

- **4. 必要时考虑检索增强**
  - 当前版本使用的是纯向量相似度（FAISS + Embedding），在大多数场景下表现良好；
  - 如果后续在「跨医学 + 心理 + 社科」这类复杂问题上仍觉得不稳，可以进一步引入：
    - 关键词检索（BM25）+ 向量检索的混合策略；
    - rerank 模型，对 Top‑N 命中文档进行更精细的重排序。
